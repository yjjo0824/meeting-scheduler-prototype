# IMPLEMENTATION_SPEC.md — 화면 구현 보조 명세 (v2)

## Naming Rule
- UI 문구와 설명은 한국어로 작성한다.
- 파일명, 컴포넌트명, 함수명, 상태값은 영어로 작성한다.
- 한국어 UI 문구는 seed.json 또는 SPEC.md의 표현을 우선한다.

## 0. 문서 우선순위
1. SPEC.md가 정본이다.
2. seed.json은 데모 데이터의 유일한 출처다.
3. verify_seed.py는 계산 엔진 정답지다.
4. CLAUDE.md는 Claude Code 작업 규칙이다.
5. 이 문서는 화면/상태/구현 순서를 줄인 보조 명세다.

SPEC.md와 충돌하면 반드시 SPEC.md를 따른다.

## 1. 절대 구현 원칙
- seed.json 값을 하드코딩하지 말고 항상 로드한다. 양보 비용 가중치도 seed.cost_weights에서 로드한다.
- seed.people은 **배열**이다. 접근은 `people.find(p => p.id === 'doyun')` 방식으로 한다(`seed.people.doyun` 금지).
- 계산 엔진은 일반 규칙으로 동작해야 한다. 시드는 스트레스 테스트다.
- 엔진 결과는 python3 verify_seed.py의 결과와 일치해야 한다.
- 후보 생성은 **최소 제외 집합 통합 규칙**(SPEC R1): 필수 전원 하드 프리 슬롯만 후보, 제외 집합 = 그 슬롯을 하드 차단하는 선택 참석자(자동 결정). 전원 참석/1명 제외/다중 제외를 분기하지 말고 한 공식으로 계산한다.
- 후보군 그룹핑은 **표시 레이어에서만** 한다(SPEC R2). 그룹 키 = (제외 집합, 선호 미반영 사람 집합). 엔진은 슬롯 단위로 전부 계산한다.
- seed.schedule_display의 날짜는 **표시 전용**이다. 엔진은 요일×시각으로만 계산하고, 날짜 필드를 계산 로직에 절대 넣지 않는다. 요일→날짜 매핑은 window_start_date(월요일 보장, verify가 검증)에서 파생한다.
- cost는 엔진 내부 랭킹 값이다. UI에는 숫자 대신 포기 항목 문구로 표현한다("서연 님 선호 1건 양보").
- 상태는 메모리에만 둔다. localStorage, 서버 저장 금지.
- 실제 캘린더 API, 로그인, 회원가입, 재조율 상세 플로우는 구현하지 않는다.
- 사유/원문은 주최자나 다른 참여자에게 노출하지 않는다.
- 후보에는 반드시 "무엇을 포기하는지"를 명시한다.
- 확정은 시스템이 자동으로 하지 않고 주최자가 선택한다.
- 기준 환경은 데스크톱이다. 좁은 화면에서는
  "더 넓은 화면에서 보면 참여자들의 시간 조건을 한눈에 비교하기 쉬워요"
  안내 배너를 유지하되, 주최자의 핵심 과업인 현황 확인·조건 조회·
  필수/선택 변경·리마인드·후보 비교·확정을 수행할 수 있는 모바일 뷰를
  제공한다(SPEC §6의 축약 대응). 투어의 기준 경험은 데스크톱이다.

## 5. Screen 2 — ParticipantPhoneFrame
목적:
- 도윤이 말로 조건을 입력하고, 시스템이 칩으로 정리한 결과를 확인한다.

필수 구조 (5층: 맥락 → 확인 → 진술 → 검수 → 제출):
- 맥락 헤더: 요청자, 회의명, 길이, 범위, 응답 기한
- 캘린더 프리필: 월 17시 개인 일정 카드("캘린더 · 탭해서 정정" 라벨) + 일정 없는 날은 접기("화–금 일정 없음")
- 자연어 입력 질문: "여기 없는 일정이나 피하고 싶은 시간이 있나요?" — 빈 제출 허용(캘린더만으로 응답 완료)
- 입력값: `people.find(p => p.id === 'doyun').response.raw`
- 칩 검수: 문구 "이렇게 이해했어요 — 탭해서 고칠 수 있어요"
  - [불가] 수요일 14,15,16,17 · 외부 미팅
  - [병합] 월요일 17 · 캘린더에 이미 있어요
  - [회피] 금요일 17 · 웬만하면
  - 칩 탭 → 성격 전환(불가↔회피) / 시간 수정 / 삭제 → 즉시 재계산(자유 모드)
- 신뢰 문구(제출 버튼 **바로 위**): "주최자에게는 조건만 전달돼요. 이유와 원문은 보이지 않아요."
- 버튼: 응답 보내기

상태:
- 제출 완료: 요약 + "회의가 확정되기 전까지 언제든 수정할 수 있어요" + 수정 진입점
- 확정 후(잠김): 읽기 전용 + 버튼 "확정된 시간에 참석이 어려워졌어요"(신고 — 주최자에게 알림만, 상세 플로우 없음)
- 정정 칩: 프리필 카드 탭 → [이 시간 비어 있어요]/[옮길 수 있어요] → 카드는 삭제하지 않고 취소선 + 실행 취소. [비어 있어요] 직후에만 "사내 캘린더에서 열기" 보조 링크(모달 아님, 투어 미포함)
- 분류 실패: 시간을 못 읽은 문장은 조용히 버리지 말고 회색 미분류 칩으로 남기고 탭해서 시간 지정 요청

자연어 분류 구현:
- 투어의 도윤 입력은 seed의 chips를 정답으로 사용한다(원문 그대로일 때).
- 자유 모드의 임의 입력은 규칙 기반 파싱(요일·시간 표현 + "안 돼요/불가"=불가, "피하고 싶다/웬만하면"=회피, "옮길 수 있다"=조정가능). 파싱 실패 시 미분류 칩.

인터랙션:
- 응답 보내기 클릭 → 도윤 응답 반영 → 폰 프레임 아웃 → 주최자 화면 복귀 → 재계산

## 6. Screen 3 — TradeoffCandidates
목적:
- 전원 만족 시간이 없을 때, 포기 항목이 다른 후보를 비교하게 한다.

진입 조건:
- 도윤 응답 완료
- perfectSlots.length === 0

필수 표시:
- "모두가 완벽히 만족하는 시간은 없어요"
- "포기하는 항목이 다른 후보를 계산했어요"
- 후보는 **후보군 카드**로 표시(그룹 키 = 제외 집합 × 선호 미반영 사람 집합). 카드 정보는 2층:
  - 1층(집계, 사람 단위, 분모는 항상 전체 초대 인원): "참석 6/6" 등
  - 2층(구체 포기, 주체 명시): "서연 님 선호 1건 미반영" — 건수(칩 단위)는 여기서만, cost 숫자는 어디에도 비노출
- 추천 후보군 (강조):
  - 금요일 13시 · 참석 6/6
  - 포기 항목: 서연 님 선호 1건 양보 ("이 안은 서연 님 회피 1건과 충돌")
  - CTA: 이 시간으로 확정
- 대안 후보군 1 (접힌 상태, 펼쳐서 비교):
  - 수요일 14~17시 · 참석 5/6 · 기본 선택 수 14(그룹 내 최이른 슬롯) · 시간 3개 노출 + 나머지는 "n개 더 보기"(단, 숨는 게 1개뿐이면 4개 전부 노출 — 시드가 이 케이스)
  - 포기 항목: 선택 참석자 도윤 제외 · 남는 참석자 선호 전부 충족 · 더 이른 날짜
- 대안 후보군 2 (접힌 상태):
  - 월요일 17시 · 참석 4/6
  - 포기 항목: 선택 참석자 도윤·수아 제외
- 그룹 내 시각 선택 가능, 원탭 확정은 현재 선택된 시각을 확정
- 미응답자가 있으면 카드에 "잠정" 배지 승계

자유 모드에서 필요한 기능:
- 참석자 필수/선택 변경 → 즉시 재계산 (후보 카드에서 분류 확인·정정 가능)
- 칩 수정/삭제 → 즉시 재계산
- "콕 집어 다시 묻기" 진입점(리마인드 채널로 특정 질문 발송 — 진입점만, 상세 플로우 없음)
- 전원 만족 슬롯이 생기면 트레이드오프 화면 대신 "모두 괜찮은 시간이 있어요. [요일] [시간], 확정할까요? (다른 시간 n개)" 한 줄 모드 표시 — 포기 0건 후보군의 특수 케이스로 구현(별도 분기 최소화)
- 필수 참석자 제약이 겹쳐 후보 슬롯이 0개가 되면 빈 상태 표시: "가능한 시간이 없어요 — 필수/선택 분류나 조건을 조정해보세요"

## 7. Screen 4 — Confirmation
목적:

- 전원 참석 가능한 시간 기준으로 정해졌어요 (절차 투명성 수준의 근거 요약)
- 다시 조율하기 (진입점만 — 상세 플로우 없음)
- 변수가 생기면 이 조건들로 다시 계산해드려요

## 9. 엔진 필수 결과
가중치: seed.cost_weights 로드(불변식 optional_attendance > optional_preference — 엔진에서 assert). 후보 생성 = 최소 제외 집합 통합 규칙(SPEC R1). 랭킹 = 양보 비용 합계 오름차순 → 동률 시 이른 날짜·시간. 그룹핑은 표시 레이어(그룹 키 = 제외 집합 × 선호 미반영 사람 집합, 그룹 내 비용 동일 assert).

도윤 응답 전 (미응답 디폴트: 캘린더 확정 일정만 반영, 선호 없음):
- perfect: 수 14, 수 15, 수 16, 수 17
- 추천: 수 14 / 상태: 잠정 — 도윤 미응답, 확정 일정 기준

도윤 응답 후 (정본: seed.expected.candidate_groups_post):
- perfect: 없음
- 후보군 3개 (랭킹 순):
  1. 금 13 · 제외 없음 · 서연 선호 미반영 (cost 2, 내부값) — 추천
